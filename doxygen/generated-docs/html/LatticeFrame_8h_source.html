41 // A LatticeFrame is a ModelElement that provides a
42 // coordinate system for a subsection of the accelerator
43 // lattice. A lattice frame is located on the accelerator
44 // geometry at a specified position s. LatticeFrames can be
45 // nested in a frame hierachy, but may not partially
46 // overlap. A LatticeFrame may represent the frame for a
47 // single component (see class ComponentFrame), or may be
48 // constructed from a contiguous sequence of LatticeFrames
49 // (sub-frames).
50 //
51 // Since each LatticeFrame contains a section of the
52 // lattice, it represents the AcceleratorGeometry for that
53 // section (which forms a sub-geometry of the total lattice
54 // geometry). A LatticeFrame can therefore provide all the
55 // geometry transformation functions within the its local
56 // s-frame. Sub-frames of a LatticeFrame are located at
57 // specific (local) s positions on that geometry.
58 //
59 // LatticeFrames can be rotated and translated.
60 // Rotating/translating a LatticeFrame rotates/translates
61 // all the associated sub-frames. LatticeFrame provides
62 // methods for accessing the various coordinate
63 // transformations between various frames in the hierachy.
64 // Two types of coordinate transformations are
65 // distinguished:
66 //
67 // - Frame Transformations: if the location of a Lattice
68 // Frame on a super-frames geometry is s, then the frame
69 // transformation to that super-frame is defined as the
70 // coordinate transformation between the LatticeFrame's
71 // local coordinate frame, and the coordinate frame defined
72 // on the super-frame's geometry at s. Note that if no
73 // rotations have been applied to any of the frames in the
74 // hierachy (up to the specified super-frame), then the
75 // frame transformation is always the identity.
76 //
77 // - Physical Transformations: The physical transformation
78 // from a LatticeFrame to a specified super-frame is the
79 // total coordinate transformation between the local frame
80 // of the LatticeFrame, and the local frame of the
81 // super-frame. This transformation represents the physical
82 // 3-D location and orientation of the LatticeFrame's local
83 // frame in the local frame of the super-frame. In general
84 // it is not the identity, even when no transformations
85 // have been applied.
86 //
87 // Frame Transformations are most important during typical
88 // beam dynamics tracking operations, while Physical
89 // Transformations can be used for surveying.
