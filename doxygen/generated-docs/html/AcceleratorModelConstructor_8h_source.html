28 // Responsible for constructing an AcceleratorModel object.
29 // The nested frame hierachy is constructed internally
30 // using a frame stack: each call to NewFrame(aFrame)
31 // pushes the current frame onto the stack, and makes the
32 // new frame the current frame. A subsequent call to End
33 // Frame(aFrame) causes the current frame (aFrame) to be
34 // popped from the stack. At any time between calls to New
35 // Frame and EndFrame can AppendComponent be called.
