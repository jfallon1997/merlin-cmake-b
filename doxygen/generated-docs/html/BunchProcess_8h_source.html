26 // A single process which is applied to a Bunch object
27 // during tracking. A Process can represent any physical
28 // (eg. particle transport) or abstract mechanism (eg.
29 // output) which may be applied to a bunch at a specific
30 // AcceleratorComponent. Concrete processes may be generic
31 // to all Bunch model types , or may be specific to a
32 // concrete bunch model (see for example ParticleBunch and
33 // ParticleBunchTransortProcess).
34
